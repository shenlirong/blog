Interface和type的区别

‌Interface与Type的核心区别‌在于：interface支持声明合并和继承机制，主要用于规范对象结构；而type具备更强的类型组合能力，适用于联合类型、元组等复杂类型定义。

‌声明合并特性‌

  • ‌interface‌：允许重复声明同名接口，自动合并属性与方法（示例：两个Person接口的name和age属性会合并）。‌‌‌‌
  • ‌type‌：重复定义同名类型别名会导致编译错误，无法合并。‌

‌扩展方式差异‌

  1. ‌接口继承‌。
    • interface通过extends实现继承（如Dog extends Animal）。
    • type通过交叉类型&组合对象（如Dog = Animal & { bark() }）。
  2. ‌类实现机制‌。
    • interface可直接被类implements实现。
    • type定义的对象类型虽然也能被类实现，但无法使用extends继承。

‌适用范围对比‌

  1. ‌interface适用场景‌。
    • 定义需要扩展的对象结构（如API请求参数类型）。
  2. ‌type适用场景‌。
    • 定义联合类型（如ID = string | number）。
    • 创建元组类型（如Coordinates = [number, number]）。
    • 组合交叉类型（如Person = {name} & {age}）。
关键区别总结
特性typeinterface
语法使用 = 定义，支持任意类型使用 {} 定义，仅描述对象类型
扩展性通过交叉类型 & 组合类型通过 extends 继承接口
声明合并不支持，重复定义会报错支持同名接口自动合并
适用场景联合类型、元组、条件类型、映射类型等复杂操作面向对象设计、类实现、对象结构定义
类实现无法直接被类 implements（除非定义为对象类型）可直接被类实现
类型推导支持 typeof 动态推导变量类型不支持
