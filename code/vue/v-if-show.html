v-if 和 v-show的区别

‌v-if 和 v-show 的核心区别在于渲染机制‌：v-if 通过条件判断动态创建/销毁 DOM 元素，而 v-show 仅通过 CSS 的 display 属性切换元素可见性。主要使用场景建议：频繁切换（如选项卡）用 v-show，初始渲染条件不稳定或需减少 DOM 节点时用 v-if。

‌核心区别对比‌

  1. ‌DOM 操作机制‌。

    • v-if：条件为假时完全移除 DOM 元素，销毁组件实例及事件监听。
    • v-show：仅修改元素的 display: none 样式，DOM 元素始终存在。‌‌

  2. ‌生命周期影响‌。

    • v-if 切换时会触发组件的 created/mounted 等完整生命周期钩子。
    • v-show 初始化后不再触发生命周期变化，仅执行一次渲染。

  3. ‌模板语法支持‌。

    • v-if 支持 <template> 标签和配套的 v-else-if/v-else 语法。
    • v-show 无法与 v-else 联用，也不支持 <template>‌‌

‌性能特征与使用场景‌

‌优先使用 v-show 的情况‌

  • ‌高频切换场景‌：如选项卡切换、折叠面板（切换成本更低）。
  • ‌需要保留组件状态‌：如表单输入内容的缓存（DOM 未被销毁）。

‌优先使用 v-if 的情况‌

  • ‌初始化条件不稳定‌：如权限校验未完成时避免渲染敏感内容。
  • ‌需要触发生命周期回调‌：如组件显示时才需执行的接口请求。
  • ‌优化首屏性能‌：初始不显示的元素可减少 DOM 节点数量。‌‌

‌进阶使用建议‌

  • ‌组合使用策略‌：表格行渲染时，对总容器用 v-if 控制显隐，行内元素用 v-show 处理局部切换。
  • ‌SSR 场景注意‌：服务端渲染时 v-show 可能引发 hydration 错误，优先使用 v-if。
  • ‌性能陷阱提示‌：频繁切换的复杂组件使用 v-if 可能导致页面卡顿，需用 keep-alive 优化。‌‌
